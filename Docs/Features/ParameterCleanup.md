# パラメータ整理（未使用パラメータ削除）

プロジェクト内の削除可能なパラメータを一覧表示し、**同名パラメータの特定**・**値の有無の確認**・**不要（未使用）パラメータの削除**を行う機能です。

長年運用したモデルには、他プロジェクトからのコピーやファミリ読み込み、共有パラメータの追加などで、**同じ名前のパラメータが重複**したり、**値が一度も入力されていない不要なパラメータ**が溜まっていきます。本機能はそれらを洗い出し、まとめて削除できます。

## 対象パラメータ

「ユーザーが削除できるすべてのパラメータ」を対象とします。

| 種別 | 内容 |
|------|------|
| プロジェクト | 非共有のプロジェクトパラメータ |
| 共有 | 共有パラメータ（プロジェクトにバインド済み／ファミリ用に読込済み） |
| グローバル | グローバルパラメータ |

※ Revit 標準の組み込み（Built-in）パラメータは削除できないため、一覧には表示されません。
※ ファミリ内部のパラメータは別ドキュメント（ファミリ）側の管理となるため対象外です。

## 使い方

1. リボン **28 Tools ＞ パラメータ ＞ パラメータ整理** をクリックします。
2. ダイアログにプロジェクト内の削除可能なパラメータが一覧表示されます。
   - **同名のパラメータ**はオレンジ色で強調表示されます。
   - 「同名のみ」チェックで重複パラメータだけに絞り込めます。
3. **「値の有無を確認」** ボタンを押すと、各パラメータに値が入った要素が1つでもあるかを判定します。
   - 判定中は進捗バーが表示され、**「中止」** でいつでも中断できます。
   - 結果は一番右の **「値」列** に「値あり」（緑）／「空（未使用）」（赤）で表示されます。
   - 確認完了後、下部に **「値あり ◯件 / 空 ◯件」** のサマリーが表示されます。
4. 削除したいパラメータの「選択」チェックを入れます。
   - **「空を全選択」** ボタンで、空（未使用）と判定されたパラメータを一括選択できます。
5. **「選択したパラメータを削除」** を押し、確認ダイアログで **はい** を選ぶと削除が実行されます。

## 大容量モデルでのフリーズ対策

Revit API はシングルスレッドのため、値の有無判定を全モデルに対して行うと処理が重くなります。本機能は次の設計でフリーズを回避しています。

- **遅延実行**: ダイアログを開いた時点ではパラメータ一覧の作成のみ（軽量）。重い値判定は「値の有無を確認」を押したときだけ実行します。
- **走査量の最小化**: 各パラメータがバインドされたカテゴリの要素だけを対象にし、カテゴリ単位で要素リストをキャッシュして再利用します。
- **早期終了**: 値が1件でも見つかった時点でそのパラメータの判定を打ち切ります。
- **進捗表示と中止**: 一定件数ごとに画面へ制御を返すため、進捗バーの更新と「中止」ボタンが常に反応します。

## 値の有無の判定について

- **文字列パラメータ**: 空白のみ・未入力は「空」と判定します。
- **ElementId パラメータ**: 無効な参照は「空」と判定します。
- **数値・整数（Yes/No 等）パラメータ**: 常に値を持つ扱いとし、安全側で「値あり」と判定します（誤って数値パラメータを削除候補にしないため）。
- **グローバルパラメータ**: 現在値を表示します（自動での未使用判定は行いません）。

## 集計表参照の表示

「**集計表参照**」列には、そのパラメータを**集計表（スケジュール）のフィールドで使用している集計表名**が表示されます（列挙時に自動取得。全要素走査ではないため軽量です）。削除前に「どの集計表で使われているか」を確認できます。

⚠️ この列が**空でも「どこからも参照されていない」とは限りません**。以下は Revit API の制約により検出対象外です：

- **ビューフィルタ**: API がフィルタ規則から参照パラメータを確実に取得できないため未対応
- **タグ・数式**: 参照情報がファミリ文書内にあり、全ファミリを開く必要があるため（重く実質不可）未対応

このため、削除前の警告ダイアログは引き続き表示されます。

## ⚠️ 削除時の注意

- 削除は **元に戻せません**（Undo 前提ではありません）。
- パラメータの値が空でも、**ビューフィルタ・集計表・タグ・数式** などから参照されている場合があります。それらを削除すると機能しなくなることがあるため、削除前に必ず内容をご確認ください。
- 心配な場合は、事前にモデルのバックアップ（別名保存）を取ってから実行してください。
