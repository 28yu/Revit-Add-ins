# アイコンデザインコンセプト

## 技術仕様

### アイコンファイル規格
- **解像度**: 96×96 ピクセル
- **DPI メタデータ**: 288 DPI（PNG の pHYs チャンク）
- **WPF 論理サイズ**: 96 ÷ (288 ÷ 96) = **32 論理ピクセル**（リボンボタン枠にちょうど収まる）
- **ファイル名規則**: `{name}_96.png`
- **フォーマット**: PNG（透過背景）

### 描画スペースの考え方
- 32×32 の論理座標系で設計し、`s(v) = v * 3` で 96px にスケール
- 座標・サイズはすべて `s()` 関数を通す（例: `s(16)` = 中央）

### 生成ツール
- **メインツール**: `Tools/IconGenerator96.html`（ブラウザで開いてプレビュー＆ダウンロード）
- **DPI 設定**: ツール内で pHYs チャンクを自動付与（288 DPI）
- **フォールバック**: Python Pillow（`dpi=(288.0106, 288.0106)` を指定）

## シート表現の設計ルール（sheet_creation / viewport 系）

シートを表現するすべてのアイコンで統一するルール。

### 縦横比
- **A版用紙の縦横比（√2:1）に準拠**
- 32 座標系での標準サイズ: **幅 26 × 高さ 18**（= 1.44:1 ≈ √2）

### 線の太さ（統一）
- シート外枠線・図枠線・表題欄線： すべて **`s(0.6)`** で統一
- コピー/ペーストの区別用ビューポート線のみ **`s(1.2)`**

### 図枠線
- シート外枠の内側 2px 内側に配置
- 色: `MID (#5C5C5C)`、太さ: `s(0.6)`

### 表題欄（右下の小さい四角）
- 図枠の右下角に配置（図枠の右端・下端と一致）
- サイズ: 幅 8 × 高さ 3 程度
- 色: `MID`（図枠線と同色）、太さ: `s(0.6)`（図枠線と同太さ）

### コピー vs ペーストの区別
- **コピー**: 破線（`setLineDash([s(2.5), s(1.5)])`）
- **ペースト**: 実線
- 両方とも線幅は同じ `s(1.2)`

## 各アイコンのデザイン概要

### 通り芯・レベル パネル

#### `both`（両端表示）
- 2本の縦線 + 2本の横線（格子状）
- 各交点にバブル（白丸、黒縁 R=3）
- 線はバブルの縁から縁まで（はみ出さない）

#### `left`（左端のみ）
- 上2つ + 左2つのバブル（合計4個）
- 線の右端・下端はバブルなしで canvas 端まで伸ばす

#### `right`（右端のみ）
- 上2つ + 右2つのバブル（合計4個）
- 線の左端・下端はバブルなしで canvas 端まで伸ばす

---

### シート パネル

#### `sheet_creation`（シート一括作成）
- **3枚のシートが斜め方向に重なる**表現
- 奥から手前へ: LIGHT → MID → DARK の線色で奥行き感
- 最前面シートのみ図枠線 + 表題欄を描画
- シート配置（各 26×18）: 3枚目(6,5)、2枚目(3,8)、1枚目(0.5,11)

---

### ビュー パネル

#### `viewport_copy`（ビューポート位置コピー）
- シート1枚（A版比率）+ 図枠線 + 表題欄
- ビューポートを表す **青色破線の四角**（シート左寄り）
- 破線: `[s(2.5), s(1.5)]`

#### `viewport_paste`（ビューポート位置ペースト）
- シート1枚（A版比率）+ 図枠線 + 表題欄
- ビューポートを表す **青色実線の四角**（位置・サイズはコピーと同じ）

#### `cropbox_copy`（トリミング領域コピー）
- オレンジの四角枠 + 8点のハンドル（四隅 + 辺中点）

#### `cropbox_paste`（トリミング領域ペースト）
- 青の四角枠 + 8点のハンドル（位置・構成はコピーと同じ）

#### `view_copy`（3Dビューコピー）
- アイソメトリック立方体（淡い青系3色）、6頂点定義（Vt/Vtr/Vtl/Vc/Vbr/Vbl/Vb）
- 面: 上面 `#E0EEF8`、左面 `#B0CCE0`、右面 `#88B0CC`、辺: `#2A4A6A`
- 立方体範囲: cx=16, cy=15, dx=12, dy=7（y=1〜29, x=4〜28）

#### `view_paste`（3Dビューペースト）
- アイソメトリック立方体（濃い青系3色）、同じ6頂点定義
- 面: 上面 `#5FA8D8`、左面 `#2A7AB8`、右面 `#145890`、辺: `#0A2040`

#### `sectionbox_copy`（セクションボックスコピー）
- **キャビネット投影（四角を右上に押し出した表現）**の青破線直方体ワイヤーフレーム（12辺）+ 内部小直方体（グレー淡色3面）
- 外枠: 前面 `(2,8)-(25,30)` = 23×22、奥行き `(+5,-6)` → 後面 `(7,2)-(30,24)` = 23×22（前後面同サイズ）
- 描画順: ①後面4辺+奥行き線4辺（破線） → ②内部ソリッド → ③**前面4辺（最前面）**
  （前面を最後に描くことで「枠の中にキューブが入っている」立体感を実現）
- 破線: `[2.5, 1.5]`、線幅 `s(1.5)`、色 `#0066CC`
- 内部小直方体: 前面 `(9,14)-(18,23)` = 9×9、奥行き `(+4,-4)` → 後面 `(13,10)-(22,19)`
- 内部色: 上面 `#A8B8C8`・右面 `#7A8A9A`・前面 `#B8C8D8`

#### `sectionbox_paste`（セクションボックスペースト）
- **キャビネット投影**の青実線直方体ワイヤーフレーム（12辺）+ 内部小直方体（青濃色3面）
- 外枠: 前面・後面・奥行きパラメータは copy と同じ、実線、線幅 `s(1.5)`、色 `#2288DD`
- 描画順: copy と同じ（後面+奥行き → 内部ソリッド → 前面）
- 内部色: 上面 `#5090D0`・右面 `#2870B0`・前面 `#70B0E0`（コピーとの区別用）

#### `cropbox_copy`（トリミング領域コピー）
- **`both`（通り芯両端）パターンをベース** に青破線矩形を重ねる
- バブル半径 R=2、バブル内縁(=6)から2単位内側の矩形 (8,8)-(24,24)
- 描画順: ①グリッド線(DARK) → ②バブル → ③青破線矩形
- 破線: `[2.5, 1.5]`、線幅 `s(1.2)`（コピー・ペースト共通太さ）

#### `cropbox_paste`（トリミング領域ペースト）
- **`both`パターンをベース** に青実線矩形を重ねる
- バブル半径 R=2、矩形 (8,8)-(24,24)（コピーと同位置）
- 描画順: ①グリッド線(DARK) → ②バブル → ③青実線矩形
- 実線、線幅 `s(1.2)`

---

### 注釈・詳細 パネル

#### `room_tag`（部屋タグ一括配置）
- 表形式アイコン（罫線グリッド）
- 上部1行は横結合（通し行）、下部は3列グリッド

#### `filled_region`（塗潰し領域 分割・統合）
- 左右2つのハッチング矩形
- 下部に「←→」の矢印（分割・統合を表現）

---

### 構造 パネル

#### `beam_under_level`（梁下端色分け）
- I型梁（上部）+ 上向き矢印 + ∇FL線（下部）+ 右側3色ブロック
- 3色: ピンク / 黄 / 青

#### `beam_top_level`（梁天端色分け）
- ∇FL線（上部）+ 下向き矢印 + I型梁（下部）+ 右側3色ブロック
- 3色: ピンク / 黄 / 青

#### `fire_protection`（耐火被覆色分け）
- I型梁 + グレー塗潰し被覆 + 右側3色ブロック

#### `formwork`（型枠数量算出）
- 構造要素のアウトライン + 型枠面の可視化表現

---

### データ パネル

#### `excel_export`（EXCELエクスポート）
- 左側: Excel アイコン（緑の台形 + X マーク）
- 右側: **上向き矢印**（青）= データを出す

#### `excel_import`（EXCELインポート）
- 左側: Excel アイコン（緑の台形 + X マーク）
- 右側: **下向き矢印**（青）= データを受け取る
